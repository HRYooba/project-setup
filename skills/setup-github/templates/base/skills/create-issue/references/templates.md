# Issue テンプレート

## feat（機能追加）

```markdown
## 概要
<!-- 追加したい内容を1〜3文で簡潔に説明 -->

## 背景・目的
<!-- なぜ必要か、何を解決したいか -->

## スコープ
<!-- 今回このIssueで扱う範囲 -->

## 対象外
<!-- 今回あえてやらないこと、別Issueで扱うこと -->

## 制約・注意点
<!-- 互換性、性能、既存仕様、触ってはいけない箇所など。なければ「特になし」 -->

## 完了条件
- [ ] 条件1
- [ ] 条件2

## 検証観点
- [ ] 確認観点1
- [ ] 確認観点2

## 参考
<!-- 関連Issue / PR / 仕様書 / 参考リンク。なければ「なし」 -->
```

## fix（バグ報告）

```markdown
## 概要
<!-- バグの簡潔な説明 -->

## 背景・目的
<!-- 影響範囲や優先度の理由。なければ省略可 -->

## 再現手順
1. 手順1
2. 手順2

## 期待する動作
<!-- 本来どうなるべきか -->

## 実際の動作
<!-- 現在どうなっているか -->

## スコープ
<!-- 今回このIssueで扱う範囲 -->

## 対象外
<!-- 今回やらないこと。なければ「なし」 -->

## 制約・注意点
<!-- 再現条件、暫定回避策、触ってはいけない箇所など。なければ「特になし」 -->

## 完了条件
- [ ] 不具合が再現しない
- [ ] 影響範囲の確認が完了している

## 検証観点
- [ ] 再現手順ベースで修正確認できる
- [ ] 関連する既存機能に退行がない

## 参考
<!-- ログ、スクリーンショット、動画、関連Issueなど。なければ「なし」 -->
```
